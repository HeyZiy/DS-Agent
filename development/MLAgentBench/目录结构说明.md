# MLAgentBench 目录结构说明

本文档说明当前目录（`development/MLAgentBench`）下各文件与文件夹的作用。

## 顶层文件

- `LLM.py`：统一封装大模型调用逻辑（如 `complete_text`、`complete_text_fast`），并处理 token 统计、日志记录等。
- `__init__.py`：Python 包初始化文件，标记该目录可被作为模块导入。
- `environment.py`：实验运行环境管理（任务加载、动作执行、日志轨迹、环境生命周期）。
- `eval.py`：评估相关入口或工具脚本。
- `high_level_actions.py`：高层动作定义（例如“制定实验计划”“总结进展”等复合动作）。
- `low_level_actions.py`：低层动作定义（读写文件、执行脚本等基础动作）。
- `mlagent_runner.py`：与主运行流程相关的辅助 runner（通常用于不同运行方式/封装）。
- `prepare_task.py`：任务准备逻辑（准备数据、脚本、工作目录等）。
- `retrieval.py`：检索模块（case 检索与重排，默认会涉及嵌入模型加载）。
- `runner.py`：主入口脚本（参数解析、环境初始化、选择 agent 并启动）。
- `schema.py`：数据结构/类型定义（动作、异常、通信结构等）。

## 顶层目录

- `agents/`：不同 Agent 策略实现。
- `benchmarks/`：各任务基准数据与脚本模板。
- `logs/`：每次运行产生的日志目录（按时间戳分组）。
- `workspace/`：每次运行的工作区快照（按时间戳分组）。
- `__pycache__/`：Python 运行时生成的字节码缓存，可忽略。

## agents/（Agent 实现）

- `agent.py`：基础 Agent 抽象与通用执行框架。
- `agent_research.py`：ResearchAgent 实现（研究型迭代流程）。
- `dsagent.py`：DSAgent 实现（当前默认 agent 类型）。
- `agent_langchain.py`：基于 LangChain 的 agent 实现。
- `agent_autogpt.py`：AutoGPT 风格 agent（可选依赖，未装依赖可能无法导入）。
- `utils.py`：Agent 相关工具函数。
- `__init__.py`、`__pycache__/`：包初始化与缓存目录。

## benchmarks/（任务基准）

该目录下每个子文件夹对应一个任务，例如：
- `feedback/`、`textual-entailment/`、`wild-blueberry-yield/`、`spaceship-titanic/` 等。

典型任务结构（以 `feedback/` 为例）：
- `scripts/`：任务说明与准备标记（如 `research_problem.txt`、`prepared`）。
- `env/`：任务运行所需文件（如 `train.py`、`train.csv`、`test.csv`、`submission.py`、`answer.csv`）。

## logs/（运行日志）

按运行时间戳分目录，例如：
- `20260320-142626/`
- `20260320-152122/`

单次运行日志通常包含：
- `agent_log/main_log`：Agent 决策与动作主日志。
- `env_log/error.txt`：异常堆栈。
- `env_log/trace.json`：结构化动作轨迹。
- `env_log/overall_time.txt`：总耗时统计。
- `env_log/tool_logs/`、`env_log/traces/`：工具级细粒度日志与轨迹。

## workspace/（运行工作区）

同样按时间戳分目录，每次运行会复制/生成任务工作副本，例如：
- `workspace/20260320-142626/feedback/`

工作区中通常包含：
- 任务数据文件（`train.csv`、`test.csv`）。
- 任务脚本（`train.py`、`submission.py`）。
- 运行产物（`answer.csv`）。
- `backup/`：脚本编辑回滚或历史备份。

## 备注

- `benchmarks/` 是“基准模板与原始任务资源”；`workspace/` 是“某次运行时的可修改副本”。
- 如果网络不可用，`retrieval.py` 中默认模型（如 `BAAI/llm-embedder`）可能导致检索步骤失败。
